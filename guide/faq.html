常见问题 FAQ ​

收集日常接入中最常遇到的问题，并提供排查建议。欢迎在 issue 中继续反馈。

1. 按需引入后组件找不到？ ​

  • 确认使用了 unplugin-auto-import / babel-plugin-import 等插件
  • 检查别名是否指向构建产物（如 Vite resolve.alias）
  • 如果直接引用源码，需确保 TypeScript/打包器包含 responsive/src 目录

2. Vue 2 提示 composition-api 未安装 ​

  • 运行 npm install vue-demi @vue/composition-api
  • 在入口 main.js 中执行 Vue.use(CompositionApi) 后再安装插件
  • 对于 vite-plugin-vue2，确保 plugins: [createVuePlugin()]

3. React 项目报 TS 类型冲突 ​

  • 根目录的 tsconfig 应排除 Vue 专用文件："exclude": ["responsive/**/*.vue"]
  • 建议为 React demo 使用独立 tsconfig.json
  • 若冲突来自 JSX 命名空间，确认 Babel / TS 配置未混用 Vue JSX 插件

4. Masonry 滚动卡顿 ​

  • 启用 virtual
  • 调整 overscanForward，默认 160，可逐步增加到 240~320
  • 使用性能工具观察是否存在大图片未懒加载
  • 若数据量极大，可拆分为多个 Masonry 实例或分页

5. 首屏变量不同步导致闪烁 ​

  • SSR 场景应开启 suppressInitialDomWrite
  • 服务端注入 initialSnapshot 并在客户端调用 flushDom()
  • 确保首屏容器宽度与服务端一致，避免 hydration mismatch

6. 旧浏览器不支持 ResizeObserver ​

  • 调用 ensureResponsivePolyfills()
  • 若使用按需加载框架，确认动态 import 被正确打包
  • 也可以手动在入口引入 polyfill 包

7. 主题变量没有生效？ ​

  • 主题对象通过 ThemeProvider 或插件 theme 选项传入
  • 检查变量名是否与 themeVar("colors.brand") 对应
  • 样式覆写时可使用 styled(Component) 保留 className

8. 如何同时支持 PC、移动和大屏？ ​

  • 设置合理的 designWidth（PC: 1440、移动: 375、大屏: 1920）
  • 使用 ResponsiveValue 对象分断点设置不同属性
  • 大屏下结合 initialSnapshot 固定宽度，避免缩放受浏览器影响

9. 运行 demo 提示样式路径找不到 ​

  • 示例项目通过相对路径引入 responsive/src/styles.css
  • 如切换到包发布版本，请修改为 @tlgf/responsive/styles.css

10. 如何扩展更多自定义组件？ ​

  • 使用 styled(YourComponent) 包裹，即可复用响应式变量
  • 或者使用 createResponsiveManager 自行管理状态，再结合 useResponsiveManager() 暴露给子组件

如仍有疑问，可在仓库 issue 或讨论区附上最小复现、项目配置与浏览器信息，我们会协助定位。
