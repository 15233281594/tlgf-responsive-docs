样式与响应式指南 ​

本章节聚焦 styled API、CSS 变量体系与多端布局策略，帮助你快速搭建 PC / 移动端 / 大屏统一的视图层。

styled 基础用法 ​

Vue 组合式示例 ​

ts
import { styled } from "@tlgf/responsive";

const Card = styled.div`
  background: var(--rs-surface, #fff);
  border-radius: 16px;
  padding: calc(12px * var(--rs-scale));
`
  .attrs({ role: "article" })
  .withConfig({ displayName: "DemoCard" });

// Vue 3 组件中使用
<Card>
  <h3>&#123;&#123; state.breakpoint &#125;&#125;</h3>
</Card>

React 示例 ​

tsx
import { styled, useResponsive } from "@tlgf/responsive/react";

const Card = styled.div`
  background: var(--rs-surface, #fff);
  border-radius: 16px;
  padding: calc(12px * var(--rs-scale));
`;

export function Panel() {
  const rs = useResponsive();
  return (
    <Card data-breakpoint={rs.breakpoint}>
      {rs.viewportWidth}px
    </Card>
  );
}

attrs 与 withConfig ​

  • attrs(factory)：按需合并静态 / 动态属性，可访问 props + 响应式上下文
  • withConfig({ shouldForwardProp, componentId, displayName })：自定义过滤属性、覆盖 class 前缀
ts
const Button = styled.button`
  min-width: 120px;
  font-size: calc(14px * var(--rs-font-scale));
`
  .attrs((props, ctx) => ({
    type: "button",
    "data-breakpoint": ctx.breakpoint,
  }))
  .withConfig({
    shouldForwardProp: (key) => key !== "variant",
  });

CSS 变量速查 ​

变量含义典型用途
--rs-scale当前缩放倍数（auto clamp by min/max）控制 paddings、字号、间距
--rs-font-scale字体独立缩放针对大屏或老人机 UI 提升可读性
--rs-breakpoint命中的断点 key，例如 md搭配 data-breakpoint、条件样式
--rs-viewport-width容器/视口宽度计算百分比布局、瀑布流列数
--rs-device-pixel-ratioDPR图片选择、Canvas 渲染
自定义如 --rs-color-primary主题色通过 ThemeProvider 注入

辅助函数：

  • cssVar("spacing-lg") → var(--rs-spacing-lg)，用于组合变量命名
  • themeVar("colors.brand") → 访问主题对象下的嵌套属性

主题与多配色 ​

Vue：

ts
import { ThemeProvider } from "@tlgf/responsive";

<ThemeProvider value=&#123;&#123; colors: { brand: "#2563eb" } &#125;&#125;>
  <App />
</ThemeProvider>

React：

tsx
import { ThemeProvider } from "@tlgf/responsive/react";

<ThemeProvider value=&#123;&#123; colors: { brand: "#2563eb" } &#125;&#125;>
  <App />
</ThemeProvider>

主题会同步到 styled 模板，可结合 attrs 做到类型提示：

ts
const Title = styled.h2<{ tone?: "primary" | "secondary" }>`
  color: ${({ tone = "primary" }, _, theme) =>
    tone === "primary" ? theme.colors.brand : theme.colors.muted};
`;

多端布局策略 ​

桌面端（PC） ​

  • designWidth: 1440 搭配多列 ResponsiveGrid
  • 使用 columnWeights 控制 Masonry 列宽，比如 [1.2, 1, 1]
  • 利用 styled + cssVar("gutter") 统一间距

移动端（H5） ​

  • 将 minScale 设为 0.85，避免过窄屏幕过度缩小
  • ResponsiveFlex + flex-wrap 实现流式布局
  • Masonry：减少 overscanForward，优先首屏速度

大屏看板 / 可视化 ​

  • initialSnapshot: { viewportWidth: 1920 } 固定基准
  • ResponsiveStack 嵌套多个区域，配合 ThemeProvider 输出夜间模式
  • 使用 withConfig({ componentId: "dashboard" }) 统一 class 前缀，方便自定义样式覆写

响应式工具集 ​

  • useResponsive() / useResponsiveContext()：在组件内拿到实时状态
  • ResponsiveBreakpoint 枚举类型：约束断点 key
  • observeResize()（核心 runtime 暴露）：针对自定义组件监听尺寸，用于图表重绘

以上内容覆盖了样式系统的常见能力，更多性能或 tree-shaking 相关内容请继续阅读其他章节。
